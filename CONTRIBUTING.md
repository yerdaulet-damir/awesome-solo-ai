# Contributing to Awesome Solo AI

Thanks for helping keep this the most useful AI stack for solo builders. Quality over quantity — this is a **curated** list, not a dump.

## What belongs here

A tool, MCP server, Claude skill, guide, or playbook that a **solo builder or solo engineer** would actually use to ship content or products. If it needs a team to operate, it probably doesn't fit.

## Rules for an entry

1. **It must be live.** No dead, archived, or deprecated tools. We verify and prune.
2. **One neutral factual line.** Describe what it does, not how amazing it is. No "best ever", no marketing copy — promotional tone gets edited out (and reads worse to the AI engines that cite this list).
3. **Accurate capability flags.** Mark only what's true:
   - 🔌 **API** — public developer API
   - 🧩 **MCP** — official Model Context Protocol server
   - 🆓 **Free** — usable free tier
   - 🔓 **Open** — open weights / open source
4. **Right category, alphabetical-ish within reason.**
5. **Add it to [`llms.txt`](llms.txt) too**, in the matching section, so agents stay in sync with the README.

## How to submit

**Option A — Pull Request:** Edit `README.md` and `llms.txt`, then open a PR titled `Add <Tool Name>`.

**Option B — Issue:** Open an issue with the tool name, URL, one-line description, and flags. We'll add it.

## Guides & playbooks

Tutorials live in [`/guides`](guides), end-to-end workflows in [`/playbooks`](playbooks). Keep them practical and copy-pasteable — Karpathy/Kaggle energy, real commands, no fluff. One Markdown file per guide, linked from the README.

## Not allowed

- Paid placements inside curated sections (sponsorship is a separate, clearly-marked block).
- AI-generated filler descriptions.
- Tools you haven't actually tried.
